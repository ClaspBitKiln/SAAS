## Видение Founder (дословно)

Пришла заявка → автоматическая расценка (если надо — с доставкой) → автоматический запрос у поставщиков → автоматическое создание КП или счёта → отправка клиенту.

## Почему это дифференциатор

Битрикс24/amoCRM хранят карточки — они не считают цену металлопроката и не делают КП за менеджера. Цикл «заявка→КП» у трейдера занимает часы (найти прайсы, посчитать доставку, собрать КП руками). Сократить до минут = измеримая ценность, за которую платят.

## Целевой конвейер

```
Заявка (текст из Telegram/MAX/email/звонка)
  ↓ AI-парсинг позиций (LLM: текст → строки)
Позиции: [марка/сортамент, размер, количество, ед.]
  ↓ детерминированная расценка
Цена = закупка (прайс) + маржа (правило) + доставка (тариф × тоннаж × маршрут)
  ↓ если позиции нет в прайсах
Автозапрос поставщикам (шаблон, рассылка) → сбор ответов
  ↓
КП / счёт (PDF, реквизиты, шаблон)
  ↓ ПОДТВЕРЖДЕНИЕ МЕНЕДЖЕРА (human-in-the-loop)
Отправка клиенту → follow-up задача
```

## Жёсткие принципы (из CLAUDE.md / AI_RULES)

1. **Деньги считает не LLM.** Расценка, маржа, доставка — детерминированные формулы и справочники. LLM только: парсинг свободного текста заявки, черновики писем/КП-текста, summary.
2. **Human-in-the-loop сначала.** Автомат готовит — менеджер жмёт «Отправить». Полный автопилот только после накопленного доверия (метрика: % КП, отправленных без правок).
3. **Каждый этап полезен отдельно.** Не строить весь конвейер сразу — каждый R-слайс сокращает время менеджера сам по себе.

## Этапы (после P1 из docs/102)

- **R1. Парсинг заявки** — свободный текст → структурированные позиции. Scaffold уже есть (Request parse). Ценность: заявка из мессенджера в систему за секунды.
- **R2. Расценка** — Products + прайсы + маржа + доставка → цена по позициям. Детерминированно. Ценность: расценка минуты вместо часа.
- **R3. КП/счёт PDF** — шаблон + позиции + реквизиты → документ. Ценность: КП в один клик.
- **R4. Отправка** — email v1 (SMTP), Telegram/MAX после блока Communications. С подтверждением менеджера.
- **R5. Автозапрос поставщикам** — если позиции нет в прайсе: шаблонная рассылка N поставщикам, статусы ответов.
- **R6. Сбор ответов поставщиков** — входящие цены → в расценку. Требует Communications (Inbox).

Порядок R1→R2→R3→R4 даёт «заявка→КП за минуты» без мессенджер-интеграции. R5–R6 зависят от блока Communications (Telegram+MAX, см. DECISIONS 2026-07-03).

## Метрика успеха

Время «заявка получена → КП отправлено»: сейчас (замерить у менеджера!) → цель < 15 мин. Второе: % заявок, расцененных без ручного поиска цен.
