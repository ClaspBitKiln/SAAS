# CIS support: компании из Узбекистана, Казахстана, Киргизии

Статус: BACKLOG (Founder, 2026-07-04: «узбекистан казахстан киргизия»). После Tasks P1.

## Проблема
Поле `inn` валидируется как РФ (10/12 цифр); автозаполнение — только ЕГРЮЛ (DaData). Контрагенты СНГ сейчас создаются без ИНН, всё руками.

## Налоговые идентификаторы по странам

| Страна | Идентификатор | Формат |
|--------|---------------|--------|
| RU | ИНН | 10 (юр) / 12 (ИП/физ) цифр |
| UZ | СТИР (ИНН) | 9 цифр |
| KZ | БИН (юр) / ИИН (физ) | 12 цифр |
| KG | ИНН | 14 цифр |

⚠ KZ БИН = 12 цифр — коллизия с РФ ИНН-12. Без поля «страна» не различить → нужен `country`.

## Дизайн (минимальный)
1. Company: + `country` (ISO-2: RU/UZ/KZ/KG, default RU) + переименовать смысл `inn` → «налоговый номер» (колонку не переименовываем, только label в UI).
2. Валидация по стране (regex per country), unique per (org, country, inn).
3. UI: select страны перед налоговым номером; label меняется (ИНН/СТИР/БИН); кнопка «Заполнить» активна только для RU (v1).
4. Автозаполнение v2 (исследовать): UZ — orginfo.uz / new.soliq.uz; KZ — pk.uchet.kz / stat.gov.kz API; KG — salyk.kg. Бесплатных стабильных API уровня DaData может не быть — проверять при взятии в работу.
5. counterparty-check (inn-bot): покрывает только РФ — для СНГ риск-скоринг отдельная тема, не смешивать.
