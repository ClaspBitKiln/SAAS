# ADR-004 — Modular Monolith

**Статус:** Принято (2026-06-21). Детализирует ADR-001.

## Решение
- No microservices. Один `apps/api`.
- Bounded contexts = NestJS-модули внутри монолита.
- Межконтекстное взаимодействие — через **доменные события** (не прямой импорт слоёв).
- **Outbox Pattern** для надёжной доставки событий.
- **BullMQ** для асинхронных/фоновых задач.

## Обоснование
До десятков тысяч клиентов модульного монолита более чем достаточно. Разделение на
сервисы добавляет операционную сложность без выгоды на нашем масштабе.

## Связанные
ADR-001, ADR-002, ADR-006.
