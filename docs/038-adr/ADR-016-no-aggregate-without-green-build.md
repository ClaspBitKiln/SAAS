# ADR-016 — No Aggregate Without Green Build

**Статус:** Принято (2026-06-21). Усиливает ADR-013/015.

## Решение
Ни один новый агрегат/контекст НЕ начинается, пока предыдущий не:
1. Компилируется (`pnpm build`).
2. Имеет зелёные unit-тесты.
3. Имеет зелёные integration-тесты.
4. Имеет зелёные e2e-тесты.
5. Запускается локально (Swagger стартует, ключевой эндпоинт работает).

## Почему
Сгенерированные файлы — это «скомпилированная спецификация», а не работающий модуль.
Module Completion (ADR-015 §2) считается выполненным только после реального прохождения
тестов. Правило предотвращает накопление «бумажной архитектуры», которую нельзя собрать.

## Definition of Done для Tenant (текущий агрегат)
```
Tenant compiles · unit green · integration green · e2e green · Swagger starts · POST /tenants works
```
Только после этого Tenant = DONE и можно начинать Organization.

ADR-013, ADR-015.
