# ADR-011 — Module Dependency Rules

**Статус:** Принято (2026-06-21). Предотвращает «big ball of mud».

## Разрешённое направление зависимостей (только сверху вниз)
```
shared → infrastructure → platform → auth → users → rbac → crm →
communication → tasks → calendar → telephony → automation → ai → analytics
```
Модуль может зависеть только от модулей ВЫШЕ по списку (через их публичные
интерфейсы/контракты), и общаться вбок/вниз только через события.

## Запрещено
- `AI → CRM` прямой доступ к репозиторию (только через публичный сервис/события).
- `Communication → Telephony` жёсткая связанность (только события).
- Циклические зависимости между модулями.
- Cross-module Prisma-запросы (один модуль не читает таблицы другого напрямую).

## Как обеспечивается
- Публичный API модуля — через `index.ts` (экспорт интерфейсов/DTO/событий), внутренние слои не экспортируются.
- Контроль: ESLint-правило на запрет импортов из чужих `domain/`/`infrastructure/`; ревью.
