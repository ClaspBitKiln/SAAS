# ADR-018 — Red Build Stops Development

**Статус:** Принято (2026-06-21).

## Решение
Если `build` = RED, или unit/integration/e2e = RED, то ЗАПРЕЩЕНО:
- генерировать новый код;
- начинать новый агрегат;
- менять архитектуру;
- добавлять новые зависимости.

Разрешено только: **analyze → fix → retest**.

## Почему
Не наслаивать новый код на сломанную базу. Красная сборка — это стоп-сигнал всей
разработки, а не «починим потом». Совместно с ADR-016 гарантирует, что система всегда
в собираемом состоянии.

## Связанные
ADR-013, ADR-016, ADR-017.
