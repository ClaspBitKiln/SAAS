# ADR-017 — Regression Protection

**Статус:** Принято (2026-06-21). Действует с первого green build.

## Решение
Каждый новый агрегат/контекст обязан:
1. не ломать предыдущие тесты;
2. проходить `pnpm build`;
3. сохранять backward compatibility публичного API;
4. увеличивать общее количество тестов;
5. не снижать coverage.

## Как обеспечивается
- CI прогоняет весь набор тестов (не только нового модуля) перед merge.
- Изменение публичных контрактов (REST/события) — только через версионирование (ADR-008) и запись в ADR.
- Coverage-гейт в CI (порог не ниже предыдущего).
